# Cursor 내장 브라우저 vs 외부 브라우저 비교

## 🔍 두 가지 실행 방식

### 방식 1: Cursor 내장 브라우저/프리뷰
- Cursor 에디터 내에서 직접 확인
- 코드 편집과 동시에 결과 확인 가능

### 방식 2: 외부 브라우저 (Chrome, Safari 등)
- 별도의 브라우저 창에서 실행
- 완전한 브라우저 기능 사용

## 📊 상세 비교

### Cursor 내장 브라우저/프리뷰

#### ✅ 장점
1. **편의성**
   - 코드 편집과 동시에 결과 확인
   - 별도 창 전환 불필요
   - 빠른 피드백

2. **통합 환경**
   - 에디터와 브라우저가 같은 화면
   - 코드 변경 시 자동 새로고침 가능

3. **간단한 확인**
   - 빠른 UI 확인에 적합
   - 간단한 테스트에 유용

#### ❌ 단점
1. **기능 제한**
   - 개발자 도구 제한적
   - 확장 프로그램 사용 불가
   - 일부 브라우저 API 제한

2. **성능**
   - 3D 렌더링 성능이 낮을 수 있음
   - 복잡한 애니메이션 제한적

3. **디버깅**
   - 네트워크 요청 상세 확인 어려움
   - 콘솔 로그 확인 제한적

### 외부 브라우저 (Chrome, Safari 등)

#### ✅ 장점
1. **완전한 기능**
   - 모든 브라우저 기능 사용 가능
   - 개발자 도구 완전 지원
   - 확장 프로그램 사용 가능

2. **성능**
   - 최적의 렌더링 성능
   - 3D 그래픽 최적화
   - 하드웨어 가속 활용

3. **디버깅**
   - 네트워크 탭으로 API 요청 확인
   - 콘솔에서 상세 로그 확인
   - React DevTools 사용 가능
   - 성능 프로파일링

4. **테스트**
   - 다양한 브라우저에서 테스트
   - 반응형 디자인 확인
   - 모바일 시뮬레이션

#### ❌ 단점
1. **편의성**
   - 별도 창 전환 필요
   - 에디터와 분리된 환경

2. **통합성**
   - 코드 변경 시 수동 새로고침 필요
   - (하지만 Vite HMR로 자동 새로고침 가능)

## 🎮 이 프로젝트에 미치는 영향

### 3D 주사위 게임 특성상

#### Cursor 내장 브라우저
- ⚠️ **3D 렌더링 성능**: 제한적일 수 있음

- ✅ **물리 엔진**: 완전한 기능
- ✅ **디버깅**: 개발자 도구로 상세 분석
- ✅ **성능 모니터링**: FPS, 메모리 사용량 확인

## 💡 추천 사용 방법

#### 1. 초기 개발 (UI 확인)
```
Cursor 내장 브라우저 사용
→ 빠른 피드백, 간단한 확인
```

#### 2. 기능 개발 (로직 테스트)
```
외부 브라우저 사용

```

#### 3. 3D/성능 테스트
```
외부 브라우저 필수

```

#### 4. 최종 테스트
```
외부 브라우저 + 다양한 브라우저
→ 크로스 브라우저 테스트
```

## 🔧 Cursor에서 외부 브라우저 열기

### 방법 1: 수동
1. 서버 실행: `npm run dev`
2. 브라우저에서 `http://localhost:5000` 접속

### 방법 2: 자동 (설정 가능)
```json
// .vscode/settings.json (Cursor도 VSCode 기반)
{
  "vite.open": true,
  "vite.openBrowser": "chrome" // 또는 "safari"
}
```

### 방법 3: Vite 설정
```typescript
// vite.config.ts
export default defineConfig({
  server: {
    open: true, // 자동으로 브라우저 열기
    openBrowser: 'chrome'
  }
});
```
