# 발라트로 빌드 메커니즘 이해 확인 질문

## 이해한 부분

### 1. 곱연산 시너지 (Baron + Mime)
- **이해**: 조커 A가 x1.5배, 조커 B가 "효과를 한 번 더 발동" → x1.5 × x1.5 = x2.25배
- **질문**: 주사위 게임에서 "효과를 한 번 더 발동"은 어떻게 구현되나요?
  - 예: "족보 배수를 한 번 더 적용" 같은 의미인가요?
  - 아니면 "조커 효과를 중복 적용"인가요?

### 2. 덱 압축 전략 (Triboulet)
- **이해**: 특정 카드(킹, 퀸)만 남기고 덱을 압축
- **질문**: 주사위 게임에서는 "주사위 면을 특정 값으로 고정"하는 방식인가요?
  - 예: 주사위의 모든 면을 6으로 고정 → 항상 높은 값

### 3. 칩/배수 균형 (Plasma Deck)
- **이해**: 칩이 많으면 자동으로 배수 역할도 함
- **질문**: 주사위 게임에서는 "주사위 값의 합이 일정 이상이면 추가 배수" 같은 의미인가요?
  - 예: 주사위 값 합이 20 이상이면 x1.5배

### 4. 덱 구조 활용 (Checkered Deck)
- **이해**: 특정 슈트만 있어서 플러시가 쉬움
- **이해 완료**: 주사위 게임에서도 슈트를 제한하면 Flush 빌드가 쉬움

### 5. 성장형 빌드 (Vampire + Midas, DNA + Hologram)
- **이해**: 매 판마다 영구적으로 강해짐
- **질문**: 주사위 게임에서 "영구적 증가"는 게임 세션 전체에 걸쳐 유지되나요?
  - 예: 핸드 제출할 때마다 +0.2 Mult 영구 증가

### 6. 특수 덱 활용
- **이해**: 덱 자체의 특수 규칙
- **질문**: 주사위 게임에서는 "주사위 덱의 초기 설정"을 의미하나요?
  - 예: 모든 주사위가 특정 슈트만 가짐

### 7. 낮은 숫자 활용 (Wee Joker)
- **이해**: 낮은 숫자(2)를 낼 때마다 칩 영구 증가
- **이해 완료**: 주사위 게임에서도 값 2를 제출할 때마다 보너스

### 8. 카드 재발동 (Hack, Chad)
- **이해**: 카드를 여러 번 발동
- **질문**: 주사위 게임에서 "재발동"은 어떻게 구현되나요?
  - 예: "주사위 값을 한 번 더 계산" 같은 의미인가요?
  - 아니면 "족보 배수를 한 번 더 적용"인가요?

### 9. 첫 카드 보너스 (Photograph)
- **이해**: 첫 번째 카드에 배수
- **질문**: 주사위 게임에서는 "첫 번째로 잠근 주사위"를 의미하나요?
  - 예: 첫 번째로 잠근 주사위의 값에 x2 배수

## 추가 질문

### 10. 주사위 게임 특화 질문
- **주사위 면 변경**: 인챈트로 주사위 면을 변경할 수 있으니, "덱 압축"은 주사위 면을 모두 특정 값으로 고정하는 것인가요?
- **족보 시너지**: 여러 족보 조커를 조합하면 어떻게 되나요?
  - 예: Pair 조커 + Two Pair 조커 → 둘 다 적용되나요?
- **슈트 시너지**: 여러 슈트 조커를 모두 장착하면?
  - 예: Greedy + Lusty + Wrathful → 각 슈트마다 보너스

### 11. 빌드 디자인 방향
- **목표**: 단일 조커가 아닌 "조커 조합"으로 빌드를 만들어야 하나요?
- **예시**: "Flush 빌드" = 슈트 조커 4개 + Flush 족보 조커 + 기타 시너지 조커

## 확인이 필요한 핵심 메커니즘

1. **"효과를 한 번 더 발동"**의 정확한 의미
2. **"덱 압축"**을 주사위 게임에서 어떻게 구현할지
3. **"재발동"** 메커니즘의 구현 방법
4. **"영구적 증가"**의 범위 (게임 세션 전체? 스테이지별?)
