# 웹 검색으로 150개 조커를 한번에 가져오지 못한 이유

## 🔍 웹 검색 결과 분석

### 제공된 웹 검색 결과 내용
제가 받은 웹 검색 결과에는:
- ✅ 발라트로 조커에 대한 일반적인 설명
- ✅ 일부 조커 이름들 (S+, S, A, B, C 등급별로 일부만 나열)
- ✅ 조커 시스템에 대한 설명 (희귀도, 수정자 등)
- ❌ **150개 조커의 전체 상세 리스트는 포함되지 않음**

### 실제 웹 검색 결과에서 본 것
```
- S+ 등급: Blue Print, Brainstorm, Triboulet (3개만)
- S 등급: Vampire, Cavendish, Duo, Trio, Family 등 (일부만)
- A 등급: Hiker, Fortune Teller, Rocket 등 (일부만)
- B 등급: 일부만 나열
- C 등급: 일부만 나열
```

**총 약 20-30개 조커 이름만 제공됨** (150개 중)

## ❓ 왜 전체 리스트를 가져오지 못했는가?

### 1. 웹 검색 도구의 한계
- **요약된 결과 제공**: 웹 검색 도구는 페이지의 모든 내용을 가져오는 것이 아니라 요약된 형태로 제공
- **토큰 제한**: 전체 페이지 내용을 모두 가져오면 토큰이 너무 많이 소모됨
- **중요 정보 우선**: 검색 결과는 일반적으로 중요한 정보를 우선적으로 포함

### 2. 페이지 구조 문제
- **동적 로드**: 위키 페이지가 JavaScript로 동적으로 로드되는 경우, 웹 검색이 모든 내용을 가져오지 못할 수 있음
- **여러 섹션**: 150개 조커가 여러 섹션이나 페이지로 나뉘어 있을 수 있음
- **표 형식**: 조커 리스트가 표 형식으로 되어 있다면, 웹 검색이 표의 모든 행을 가져오지 못할 수 있음

### 3. 검색 결과 형식
- **일부만 샘플링**: 웹 검색 결과는 전체 내용의 일부만 샘플링하여 제공
- **중복 제거**: 비슷한 내용은 중복 제거되어 표시될 수 있음
- **요약 우선**: 상세 리스트보다는 일반적인 설명을 우선적으로 제공

## 🔄 실제로 확인해보기

### 웹 검색 결과 재확인
제가 받은 웹 검색 결과를 다시 보면:
- 페이지의 **일반적인 설명**은 포함됨
- **일부 조커 이름들**은 포함됨
- 하지만 **150개 조커의 전체 상세 리스트**는 포함되지 않음

### 가능한 시나리오
1. **위키 페이지에 실제로 150개가 모두 나열되어 있음**
   - 하지만 웹 검색 결과에는 일부만 포함됨
   - 웹 검색 도구가 전체 내용을 가져오지 못함

2. **위키 페이지가 링크나 카테고리로 나뉘어 있음**
   - 각 조커가 개별 페이지로 링크되어 있을 수 있음
   - 웹 검색이 메인 페이지만 가져옴

3. **표 형식으로 되어 있어 파싱 어려움**
   - 조커 리스트가 HTML 표로 되어 있다면
   - 웹 검색이 표의 모든 행을 가져오지 못할 수 있음
