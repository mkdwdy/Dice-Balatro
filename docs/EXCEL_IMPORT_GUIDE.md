# 엑셀 파일로 조커 데이터베이스화 가이드

## ✅ 엑셀 파일 사용이 더 효율적인 이유

### 웹 검색의 한계
1. **부분적 정보만 제공**: 웹 검색 결과에는 전체 150개 조커의 상세 정보가 포함되지 않음
2. **구조화되지 않은 데이터**: 위키 페이지의 HTML 구조를 파싱하기 어려움
3. **여러 번 검색 필요**: 150개를 모두 수집하려면 수십 번의 검색이 필요
4. **정보 누락 가능**: 검색 결과에 일부 조커가 누락될 수 있음

### 엑셀 파일의 장점
1. ✅ **완전한 데이터**: 150개 조커를 한 번에 제공
2. ✅ **구조화된 형식**: 이미 정리된 데이터
3. ✅ **빠른 처리**: 한 번에 모든 데이터를 가져올 수 있음
4. ✅ **정확성**: 수동으로 확인한 데이터이므로 정확도 높음

## 📋 엑셀 파일 형식 제안

### 추천 컬럼 구조

| 컬럼명 | 설명 | 예시 |
|--------|------|------|
| id | 조커 고유 ID | `blueprint` |
| name | 조커 이름 (영문) | `Blue Print` |
| nameKorean | 조커 이름 (한글) | `청사진` |
| description | 효과 설명 (영문) | `Copies the ability of the rightmost Joker` |
| descriptionKorean | 효과 설명 (한글) | `가장 오른쪽 조커의 능력을 복사` |
| rarity | 희귀도 | `legendary` (common/uncommon/rare/legendary) |
| tier | 티어 등급 | `S+` (S+/S/A/B/C/D) |
| effectType | 효과 타입 | `synergy` |
| baseCost | 기본 가격 | `8` |
| sellValue | 판매 가격 | `4` |
| diceGameNotes | 주사위 게임 적용 노트 | `다른 조커 효과를 복사` |

### 최소 필수 컬럼
- name (영문 이름)
- description (효과 설명)
- rarity (희귀도)

나머지는 선택사항이며, 있으면 더 상세한 데이터베이스를 만들 수 있습니다.

## 📝 엑셀 파일 준비 가이드

### 방법 1: 간단한 형식 (빠름)
```
A열: name (영문 이름)
B열: description (효과 설명)
C열: rarity (common/uncommon/rare/legendary)
```

### 방법 2: 상세 형식 (권장)
위의 추천 컬럼 구조대로 작성

### 방법 3: 위키에서 복사-붙여넣기
발라트로 위키 페이지에서 표를 복사하여 엑셀에 붙여넣기
